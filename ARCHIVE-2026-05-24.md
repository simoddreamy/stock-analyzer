### 2. 探索历史报告系统

**需求**: 探索完成后自动保存报告，供后续查看；应跟随个股，在股票窗口查看该股所有历史。

**实现**:
- 数据库新增 `exploration_reports` 表（含 id/code/status/iterations/best_formula/best_precision 等）
- 后端 `main.py` 新增 3 个端点:
  - `GET /api/explore/reports?code=` (支持按股票代码过滤)
  - `GET /api/explore/reports/{report_id}`
  - `get_explore_report()` 单条详情
- `explorer/engine.py` 记录 `iteration_count`, `segment_u1`, `candidate_history`
- 探索结束时在 `main.py` 的 `run_explore()` 中调用 `save_exploration_report()`
- 前端 `api.js`: `getExploreReports(code)` 支持 code 参数
- 前端 `StocksView.vue`: 股票窗口头部加"历史报告"按钮，点击加载并显示该股历史记录

**API**:
```
GET  /api/explore/reports?code=000001   # 按股票过滤
GET  /api/explore/reports/{id}
```

**文件变更**:
- `backend/main.py` — 新增端点 + save_exploration_report()
- `backend/db/database.py` — 新增表 + ALTER TABLE 兼容
- `backend/explorer/engine.py` — 记录迭代数据
- `frontend/src/utils/api.js` — getExploreReports(code) 支持

---

### 3. 手动公式覆盖

**需求**: 用户可以手动设置公式，覆盖AI探索结果。

**实现**:
- 数据库新增 `formula_overrides` 表（code 为主键）
- 后端新增端点:
  - `PUT /api/formulas/{code}/override` — 保存覆盖
  - `GET /api/formulas/{code}/override` — 读取覆盖
- 前端 `StocksView.vue`: 公式卡片和"无公式"状态都加"设置公式"按钮
- modal 弹窗输入 formula_expr 和 logic_desc
- `get_opportunity_points` 优先读取 override，无则读 best_formulas

**API**:
```
PUT  /api/formulas/{code}/override      { formula_expr, logic_desc }
GET  /api/formulas/{code}/override
```

**文件变更**:
- `backend/main.py` — override 端点
- `frontend/src/views/StocksView.vue` — modal + 按钮
